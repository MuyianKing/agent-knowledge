# 单例浮层模式组件创建指南

## 一、核心概念

**单例浮层模式（Singleton Floating Layer Pattern）**：多个组件实例共享一个浮层组件，通过全局单例管理减少 DOM 节点，优化大数据量场景性能。

**适用场景**：
- 大数据量列表（如 5000 行）
- 每行包含相同类型的选择器组件（el-select、el-date-picker、el-cascader 等）
- 需要显著减少 DOM 节点数量

---

## 二、组件架构

### 1. 通用单例容器（SingletonField.vue）
**职责**：
- 管理全局单例生命周期
- 提供触发区域插槽
- 提供浮层内容插槽
- 动态定位浮层

**核心 Props**：
```javascript
field: String          // 分组标识，同 field 共享浮层
modelValue: any        // 当前值
displayValue: String   // 显示文本
placeholder: String    // 占位符
openMethod: Function   // 打开浮层的方法
```

**核心 Slots**：
```vue
<!-- 触发区域插槽 -->
<slot name="trigger" :display-value="displayValue" :placeholder="placeholder">
  <!-- 默认显示文本 -->
</slot>

<!-- 浮层内容插槽 -->
<slot :value="value" :on-change="onChange" :on-visible-change="onVisibleChange">
  <!-- 传入 el-select/el-date-picker 等 -->
</slot>
```

### 2. 适配层组件（Select.vue/DatePicker.vue）
**职责**：
- 提供具体组件的配置
- 实现自定义触发区域
- 传入正确的 openMethod

## 三、创建步骤

### Step 1: 创建全局单例容器

```vue
<script setup>
const props = defineProps({
  field: { type: String, required: true },
  modelValue: { type: [String, Number, Boolean, Array, Date], default: '' },
  placeholder: { type: String, default: '请选择' },
  displayValue: { type: String, default: '' },
  openMethod: { type: Function, default: null },
})

const emit = defineEmits(['update:modelValue'])

// 1. 全局单例管理
const GLOBAL_KEY = 'GLOBAL_SINGLE_MAP'
window[GLOBAL_KEY] = window[GLOBAL_KEY] || new Map()
const sharedMap = window[GLOBAL_KEY]

// 2. isHost 判定
let state = sharedMap.get(props.field)
const isHost = !state?.mounted
if (isHost) {
  state = { mounted: true, open: null, openMethod: null }
  sharedMap.set(props.field, state)
}

const triggerRef = useTemplateRef('trigger')
const visible = ref(false)
const value = ref('')
const pos = ref({ top: 0, left: 0, width: 200 })

let currentPick = null

// 3. 打开浮层
function open(targetEl, currentValue, onOpenReady, onPick) {
  const rect = targetEl.getBoundingClientRect()
  pos.value = { top: rect.top, left: rect.left, width: rect.width }
  value.value = currentValue
  visible.value = true
  currentPick = onPick
  nextTick(() => onOpenReady?.())
}

// 4. 触发器点击
function onClick() {
  state.open?.(
    triggerRef.value,
    props.modelValue,
    () => requestAnimationFrame(() => state.openMethod?.()),
    val => emit('update:modelValue', val),
  )
}

// 5. 只有 host 注册全局方法（关键！）
if (isHost) {
  watchEffect(() => {
    state.open = (el, val, onOpen, onPick) => open(el, val, onOpen, onPick)
  })
  watchEffect(() => {
    state.openMethod = props.openMethod
  })
}

// 6. 清理全局状态
onBeforeUnmount(() => {
  if (isHost) { sharedMap.delete(props.field) }
})
</script>

<template>
  <!-- 触发区域 -->
  <div ref="trigger" class="flex items-center w-[200px] h-[32px] border rounded cursor-pointer px-2" @click="onClick">
    <slot name="trigger" :display-value="displayValue" :placeholder="placeholder">
      <span :class="(modelValue === '' || modelValue == null) ? 'text-gray-400' : ''">
        {{ displayValue || placeholder }}
      </span>
    </slot>
  </div>

  <!-- 浮层（只有 host 渲染） -->
  <teleport v-if="isHost" to="body">
    <div class="fixed z-50"
         :style="{
           top: `${pos.top}px`,
           left: `${pos.left}px`,
           width: `${pos.width}px`,
           opacity: visible ? 1 : 0,        // 关键：防闪现
           pointerEvents: visible ? 'auto' : 'none',
         }">
      <slot :value="value" :on-change="onChange" :on-visible-change="onVisibleChange" />
    </div>
  </teleport>
</template>
```

### Step 2: 创建适配层组件

**Select.vue 示例**：
```vue
<script setup>
import ArrowDown from '~icons/ep/arrow-down'
import SingletonField from './SingletonField.vue'

const props = defineProps({
  field: { type: String, required: true },
  modelValue: { type: [String, Number, Boolean, Array], default: '' },
  options: { type: Array, default: () => [] },
  placeholder: { type: String, default: '请选择' },
})

const emit = defineEmits(['update:modelValue'])
const selectRef = useTemplateRef('select')
const labelOf = v => props.options.find(o => o.value === v)?.label ?? ''
</script>

<template>
  <SingletonField
    :model-value="modelValue"
    :field="field"
    :placeholder="placeholder"
    :display-value="labelOf(modelValue)"
    :open-method="() => selectRef?.focus()"        // 关键：传入打开方法
    @update:model-value="emit('update:modelValue', $event)"
  >
    <!-- 自定义触发区域 -->
    <template #trigger="{ displayValue, placeholder }">
      <span :class="(modelValue === '' || modelValue == null) ? 'text-gray-400 flex-1' : 'flex-1'">
        {{ displayValue || placeholder }}
      </span>
      <el-icon class="text-gray-400">
        <ArrowDown />
      </el-icon>
    </template>

    <!-- 浮层内容 -->
    <template #default="{ value, onChange, onVisibleChange }">
      <el-select
        ref="select"
        :model-value="value"
        :options="options"
        filterable
        automatic-dropdown              // 关键：自动展开
        @change="onChange"
        @visible-change="onVisibleChange"
      />
    </template>
  </SingletonField>
</template>
```

## 四、关键技术点

### 1. 全局单例存储
**❌ 错误**：在 `<script setup>` 中创建 Map
```javascript
const sharedMap = new Map()  // 每个实例都创建新 Map，所有实例都是 isHost=true
```

**✅ 正确**：使用 window 对象存储
```javascript
const GLOBAL_KEY = 'GLOBAL_SINGLE_MAP'
window[GLOBAL_KEY] = window[GLOBAL_KEY] || new Map()
const sharedMap = window[GLOBAL_KEY]  // 全局单例
```

### 2. 防止竞态条件
**❌ 错误**：所有实例都注册 openMethod
```javascript
watchEffect(() => {
  state.openMethod = props.openMethod  // 非 host 实例会覆盖为 null
})
```

**✅ 正确**：只有 host 注册
```javascript
if (isHost) {  // 关键条件
  watchEffect(() => {
    state.openMethod = props.openMethod
  })
}
```

### 3. 防止浮层闪现
**❌ 错误**：使用 v-show
```vue
<div v-show="visible" class="fixed">  <!-- 会左上角闪现 -->
```

**✅ 正确**：使用 opacity + pointerEvents
```vue
<div class="fixed"
     :style="{
       opacity: visible ? 1 : 0,
       pointerEvents: visible ? 'auto' : 'none',
     }">
```

### 4. 初始化时机
**问题**：Teleport 挂载需要时间，直接调用方法可能失败

**解决**：使用 requestAnimationFrame
```javascript
() => requestAnimationFrame(() => state.openMethod?.())
```

### 5. 清理全局状态
**必须**：组件卸载时清理，避免内存泄漏
```javascript
onBeforeUnmount(() => {
  if (isHost) { sharedMap.delete(props.field) }
})
```

## 五、注意事项

### 1. field 命名规范
- 同页面不同类型选择器使用不同 field
- 例如："name"、"date"、"city"、"status"
- 同 field 的实例共享同一个浮层

### 2. 组件暴露方法
- el-select：使用 `focus()` 方法
- el-date-picker：使用 `handleOpen()` 方法
- 其他组件：查阅官方文档确认暴露的方法

### 3. 图标导入
- Element Plus 图标通过 `~icons/ep/xxx` 导入
- 例如：`import ArrowDown from '~icons/ep/arrow-down'`

### 4. 样式隔离
- 浮层通过 Teleport 挂载到 body
- 避免 CSS scoped 影响浮层样式

### 5. 响应式更新
- 通过 `value` ref 同步当前值
- 通过 `onChange` 回调更新父组件
- 确保数据流正确

| DOM 节点 | ~50,000 | ~2 |

**关键指标**：5000 行数据，每行 1 个选择器

## 七、扩展新组件

**步骤**：
1. 创建适配层组件文件（如 Cascader.vue）
2. 导入 SingletonField 和所需图标
3. 定义 props（field、modelValue、options 等）
4. 实现 displayValue 格式化逻辑
5. 通过 slots 传入组件和触发器

**模板示例**：
```vue
<template>
  <SingletonField
    :model-value="modelValue"
    :field="field"
    :display-value="formatLabel(modelValue)"
    :open-method="() => cascaderRef?.focus()"
    @update:model-value="emit('update:modelValue', $event)"
  >
    <template #trigger="{ displayValue, placeholder }">
      <span class="flex-1">{{ displayValue || placeholder }}</span>
      <el-icon><ArrowDown /></el-icon>
    </template>

    <template #default="{ value, onChange, onVisibleChange }">
      <el-cascader
        ref="cascader"
        :model-value="value"
        :options="options"
        @change="onChange"
        @visible-change="onVisibleChange"
      />
    </template>
  </SingletonField>
</template>
```

## 八、常见问题

### Q1: 第一次点击不弹框，第二次才弹？
**原因**：非 host 实例覆盖了 state.openMethod
**解决**：添加 `if (isHost)` 条件

### Q2: 浮层在左上角闪现？
**原因**：使用 v-show 导致 popper 旧位置闪现
**解决**：改用 opacity + pointerEvents

### Q3: 页面卡死？
**原因**：每个实例创建独立的 Map
**解决**：改用 window.GLOBAL_SINGLE_MAP

### Q4: 如何调试？
**方法**：在浏览器控制台查看全局状态
```javascript
console.warn(window.GLOBAL_SINGLE_MAP)  // 查看 Map 内容
console.warn(window.GLOBAL_SINGLE_MAP.get('name'))  // 查看特定 field
```

## 九、最佳实践

1. **先测试小数据量**：先用 10 行数据验证功能正常
2. **逐步扩展**：确认功能正常后再测试大数据量
3. **使用 markRaw**：options 数据使用 `markRaw()` 避免不必要的响应式
4. **规范命名**：field 使用语义化命名
5. **清理调试代码**：提交前删除 console.warn 调试语句
